Length

The length CSS data type represents a distance value. Lengths can be used in numerous CSS properties, such as width, height, margin, padding, border-width, font-size, and text-shadow.

The length data type consists of a number followed by one of the units listed below. For some CSS properties, negative lengths are allowed.

Relative length units

Relative length units specify a length relative to another length property. Depending on the unit, this can be the size of a specific character, the line height, or the size of the viewport.

Font-relative units

  • • em: relative to the font-size of the element (e.g. if the font-size of the document is 12pt, 1em is equal to 12pt)
  • • rem: relative to the font-size of the root element (which is the html element in HTML documents)
  • • ex: relative to the x-height of the current font
  • • ch: relative to width of the “0”

Viewport-percentage units

  • • vw: relative to 1% of the width of viewport
  • • vh: relative to 1% of the height of viewport
  • • vmin: relative to 1% of viewport’s smaller dimension
  • • vmax: relative to 1% of viewport’s larger dimension

Using em

What happens when you nest em-defined font sizes inside each other? Each div inherits its font-size from its parent, giving us gradually increasing font-sizes.

Using rem

What happens when you nest rem-defined font sizes inside each other? Each div inherits its font-size from the root element, giving us the same font-sizes.

Absolute length units

Absolute length units represent a physical measurement when the physical properties of the output medium are known, such as for print layout. The absolute length units are fixed and a length expressed in any of these will appear as exactly that size.

Absolute length units are not recommended for use on screen, because screen sizes vary so much. However, they can be used if the output medium is known, such as for print layout.

Units

  • • cm
  • • mm
  • • in
  • • pc
  • • pt
